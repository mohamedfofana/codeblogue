Grunt est un module qui fonctionne côté client. Par conséquent, toutes les opérations sur console (nodejs, dos, ….) concernant le module se font en se positionnant au préalable dans le dossier client du projet.
-g:
permet de faire une installation globale. Cela veut dire qu’on pourra utiliser la commande grunt n’importe où sur la machine. Elle sera enregistrée dans les variables d’enironnement.
--save:
signifie que le module sera référencé dans le fichier package.json.

Si en lanceant grunt la commande n’est pas trouvée, faire un npm install sur le dossier ou se trouve le fichier package.json. Cela réinstallera les dépendances référencées dans le fichier

Quand le module compass est introuvable

Running "concurrent:server" (concurrent) task Warning: Running "compass:server" (compass) task

Warning: Command failed: 'compass.bat' is not recognized as an internal or external command, operable program or batch file. Use --force to continue.

Cette erreur signifie que la tâche grunt du module compass n’est pas reconnue. Il peut s’agir de la perte de l’installation du module compass durant une migration ou de l’absence de ruby dans le système.
Pour résoudre cette erreur, installer ruby si ce n’est pas déjà fait.

Télécharger la dernière version sur ce lien http://www.ruby-lang.org/en/documentation/installation/#rubyinstaller.
